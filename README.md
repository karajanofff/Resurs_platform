# SmartKutubxona AI

Ta'lim resurslarini fan mavzulariga mosligini NLP yordamida avtomatik baholovchi elektron kutubxona platformasi.

## Texnologiyalar

- Frontend: Next.js, TypeScript, Tailwind CSS, Lucide React, Recharts
- Backend: FastAPI, SQLAlchemy, JWT
- Database: PostgreSQL
- NLP: TF-IDF, cosine similarity, PDF/DOCX/PPTX/TXT matn ajratish

## Lokal ishga tushirish

1. `.env.example` faylini `.env` nomi bilan nusxalang.
2. Docker bilan:

```bash
docker compose up --build
```

3. Brauzerda `http://localhost:3000` ni oching.

## Demo loginlar

- Admin: `admin@example.com / admin123`
- O'qituvchi: `teacher@example.com / teacher123`

## Ishlaydigan sahifalar

- `/` - landing page
- `/login` - role tanlash va kirish
- `/dashboard` - login qilingan foydalanuvchini roliga qarab yo'naltiradi
- `/dashboard/admin?view=...` - admin bo'limlari
- `/dashboard/teacher?view=...` - o'qituvchi bo'limlari
- `/analysis` - oxirgi NLP natijasini ko'rsatadi

## NLP oqimi

1. Fayl yuklanadi.
2. Matn ajratiladi.
3. Matn tozalanadi.
4. Kalit so'zlar aniqlanadi.
5. Resurs va mavzu matni TF-IDF vektorlarga aylantiriladi.
6. Cosine similarity hisoblanadi.
7. Natija bazaga yoziladi.

Moslik qoidalari:

- `>= 75%`: `Mos`
- `40% - 75%`: `Qisman mos`
- `< 40%`: `Mos emas`

## Render.com ga deploy qilish

Ushbu repo Render uchun bitta Docker web service ko'rinishida tayyorlangan. Bitta service ichida:

- Next.js frontend tashqi portda ishlaydi
- FastAPI backend ichki `127.0.0.1:8000` portda ishlaydi
- frontend `/api/*`, `/uploads/*` va `/healthz` so'rovlarini backendga uzatadi

1. Loyihani GitHub repository ga push qiling.
2. Render Dashboardda `resurs-platform` web service ni oching.
3. `Manual Deploy` -> `Deploy latest commit` ni bosing.
4. Deploy tugagach saytni oching:

```text
https://resurs-platform.onrender.com
```

5. Health check:

```text
https://resurs-platform.onrender.com/healthz
```

6. Demo loginlar bilan kirishni tekshiring.

`render.yaml` ichida hozirgi bitta service uchun quyidagi Render sozlamalari berilgan:

```env
JWT_SECRET=<auto-generated>
UPLOAD_DIR=uploads
FRONTEND_URL=https://resurs-platform.onrender.com
BACKEND_URL=https://resurs-platform.onrender.com
```

Default holatda demo uchun SQLite ishlatiladi. Agar doimiy ma'lumot saqlash kerak bo'lsa, Render Postgres yarating va service Environment bo'limiga `DATABASE_URL` qo'shing.

Database migration bu demo versiyada `startup` vaqtida `create_all` bilan avtomatik yaratiladi. Katta production loyihada Alembic qo'shish tavsiya etiladi.

Render deployda persistent disk ishlatilmasa, SQLite bazasi va `uploads/` fayllari redeploy vaqtida yo'qolishi mumkin. Diplom demo uchun bu yetarli, production uchun Postgres va object storage tavsiya etiladi.
