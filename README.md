# SmartKutubxona AI

Ta'lim resurslarini fan mavzulariga mosligini NLP yordamida avtomatik baholovchi elektron kutubxona platformasi.

## Texnologiyalar

- Frontend: Next.js, TypeScript, Tailwind CSS, Lucide React, Recharts
- Backend: FastAPI, SQLAlchemy, JWT
- Database: PostgreSQL
- NLP: TF-IDF, cosine similarity, PDF/DOCX/PPTX/TXT matn ajratish

## Lokal ishga tushirish

1. `.env.example` faylini `.env` nomi bilan nusxalang.
2. Docker bilan:

```bash
docker compose up --build
```

3. Brauzerda `http://localhost:3000` ni oching.

## Demo loginlar

- Admin: `admin@example.com / admin123`
- O'qituvchi: `teacher@example.com / teacher123`

## Ishlaydigan sahifalar

- `/` - landing page
- `/login` - role tanlash va kirish
- `/dashboard` - login qilingan foydalanuvchini roliga qarab yo'naltiradi
- `/dashboard/admin?view=...` - admin bo'limlari
- `/dashboard/teacher?view=...` - o'qituvchi bo'limlari
- `/analysis` - oxirgi NLP natijasini ko'rsatadi

## NLP oqimi

1. Fayl yuklanadi.
2. Matn ajratiladi.
3. Matn tozalanadi.
4. Kalit so'zlar aniqlanadi.
5. Resurs va mavzu matni TF-IDF vektorlarga aylantiriladi.
6. Cosine similarity hisoblanadi.
7. Natija bazaga yoziladi.

Moslik qoidalari:

- `>= 75%`: `Mos`
- `40% - 75%`: `Qisman mos`
- `< 40%`: `Mos emas`

## Render.com ga deploy qilish

Tavsiya etilgan yo'l: repo ichidagi `render.yaml` orqali Blueprint deploy qilish.

1. Loyihani GitHub repository ga push qiling.
2. Render Dashboardda `New` -> `Blueprint` ni tanlang.
3. `karajanofff/Resurs_platform` repository sini ulang.
4. Render rootdagi `render.yaml` faylini o'qib quyidagilarni yaratadi:
   - `smartkutubxona-ai-backend`
   - `smartkutubxona-ai-frontend`
   - `smartkutubxona-db`
5. Blueprint create qiling va ikkala service build bo'lishini kuting.
6. Deploy tugagach frontendni oching:

```text
https://smartkutubxona-ai-frontend.onrender.com
```

7. Backend health check:

```text
https://smartkutubxona-ai-backend.onrender.com/healthz
```

8. Demo loginlar bilan kirishni tekshiring.

`render.yaml` ichida quyidagi Render sozlamalari allaqachon berilgan:

```env
DATABASE_URL=<Render Postgres connection string>
JWT_SECRET=<auto-generated>
UPLOAD_DIR=uploads
FRONTEND_URL=https://smartkutubxona-ai-frontend.onrender.com
BACKEND_URL=https://smartkutubxona-ai-backend.onrender.com
NODE_ENV=production
NEXT_PUBLIC_API_URL=https://smartkutubxona-ai-backend.onrender.com
```

Database migration bu demo versiyada `startup` vaqtida `create_all` bilan avtomatik yaratiladi. Katta production loyihada Alembic qo'shish tavsiya etiladi.

Render deployda backend persistent disk ishlatilmasa, `uploads/` fayllari redeploy vaqtida yo'qolishi mumkin. Diplom demo uchun bu yetarli, production uchun object storage tavsiya etiladi.

## Render bo'yicha eslatma

Agar service nomlarini o'zgartirsangiz, `FRONTEND_URL`, `BACKEND_URL` va `NEXT_PUBLIC_API_URL` qiymatlarini yangi Render URL lariga moslang. Hozirgi service nomlari `render.yaml` dagi URL lar bilan bir-biriga mos.
